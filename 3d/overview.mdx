## All 3D demos by category

### Characters and physics

| Demo | Description |
|------|-------------|
| `squash_the_creeps` | Beginner 3D game that accompanies the official Godot tutorial. Uses `CharacterBody3D` for the player and mobs. |
| `kinematic_character` | First-person kinematic character with jump, gravity, and camera follow. |
| `rigidbody_character` | Character controller built on `RigidBody3D` instead of `CharacterBody3D`. |
| `truck_town` | Vehicle physics with `VehicleBody3D`, trailers connected by `ConeTwistJoint3D`, and tow chains using `PinJoint3D`. |
| `ragdoll_physics` | Spawnable ragdolls with physical bones, slow-motion, and ray-cast placement. |
| `soft_body_physics` | `SoftBody3D` cloth, soft spheres, pinned points, and per-point impulse forces. |
| `physics_tests` | Automated and interactive collision and physics test suite. |
| `physics_interpolation` | Smooth movement between physics ticks using physics interpolation. |
| `platformer` | 3D platformer with camera follow and level geometry. |
| `csg` | Constructive Solid Geometry for prototype level building. |

|------|-------------|
| `navigation` | `NavigationAgent3D` pathfinding around a static scene with drawn path visualization. |

<Note>
Not every demo runs on all three renderers. Demos that rely on SDFGI or VoxelGI require the **Forward+** renderer. The Compatibility renderer supports LightmapGI and `CharacterBody3D` physics but lacks screen-space effects (SSAO, SSIL) and some post-processing features. Check each demo's `README.md` or `project.godot` for the required renderer.
</Note>

## Renderer overview

Godot 4 ships with three rendering backends. The 3D demos collectively cover all three.

| Renderer | Typical use | Supported in demos |
|----------|-------------|-------------------|
| **Forward+** | Desktop, high-end consoles | Most 3D demos |
| **Mobile** | Mobile, mid-range hardware | Select demos |
| **Compatibility** | Web, low-end hardware, OpenGL | `navigation`, `squash_the_creeps` fallback paths |

</Tip>
