The Godot 2D demo collection contains 27 standalone projects that each highlight a specific engine feature or game-development pattern. Every demo is written in GDScript, and most run on the **Compatibility** renderer so they work on the widest range of hardware. Two exceptions use the **Forward+** renderer (Glow and Physics Platformer), and two use the **Mobile** renderer (2D Particles and Physics Tests). Because the demos have no shared theme beyond being 2D, you can open any project independently and study exactly the technique you need.

## All 27 demos at a glance

<Note>
  The `physics_platformer` and `glow` demos require the **Forward+** renderer. If you open them in a project configured for Compatibility, Godot will warn you before switching renderers.
</Note>

## Language and renderer notes

All demos use **GDScript**. Two also include **.gdshader** files written in Godot's shader language (`sprite_shaders` and `screen_space_shaders`). C# ports of `dodge_the_creeps` and `pong` are available separately in the `mono/` directory of the repository.

The table below summarises renderer requirements:

| Renderer | Demos |
|----------|-------|
| Compatibility | 22 demos (the majority) |
| Mobile | `particles`, `physics_tests`, `custom_drawing` |
| Forward+ | `glow`, `physics_platformer` |

<Tip>
  If you are new to Godot, start with `dodge_the_creeps` — it is the finished version of the official "Your first 2D game" tutorial and introduces core concepts like signals, `Area2D`, and `AnimatedSprite2D` in a short, self-contained project.
</Tip>
