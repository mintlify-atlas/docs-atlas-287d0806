The Godot Demo Projects repository is a curated collection of open-source demonstration projects maintained by the Godot Engine team. Every folder containing a `project.godot` file is a fully runnable Godot project that shows you how to use a specific engine feature, implement a common game pattern, or structure a complete mini-game. Whether you are writing your first GDScript or integrating OpenXR hand tracking, there is a demo that shows you exactly how it is done.

## What the demos teach you

Each demo is designed to illustrate one or more concrete ideas rather than act as a complete game template. Reading through a demo teaches you:

- **GDScript idioms** — how to structure nodes, signals, and scripts the way the Godot team does it.
- **Engine feature usage** — the correct way to configure physics layers, use `CharacterBody2D`/`CharacterBody3D`, set up `AnimationPlayer` trees, and more.
- **Project organisation** — how a real `project.godot` file looks, how autoloads are wired up, and how scenes are split into reusable components.
- **Common game patterns** — enemy AI, pickups, score tracking, scene transitions, save/load, and input handling across keyboard, mouse, and gamepad.

The 2D platformer demo, for example, shows jump physics, slope handling, bullet spawning, enemy collision layers, split-screen multiplayer, and pixel-art rendering settings — all in a single self-contained project. The 3D Squash the Creeps demo is the finished product of the official "Your first 3D game" tutorial, giving you a reference point as you follow along with the docs.
