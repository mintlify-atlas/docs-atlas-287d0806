Godot's `Viewport` node (and its subclass `SubViewport`) renders a scene into a texture that you can display anywhere in your project—on a 3D mesh surface, inside a sprite, or composited through a full-screen shader. The eight demos in this group show every major use pattern: placing interactive GUI on a 3D quad, embedding a running 2D game inside a 3D scene, capturing the screen, scaling for DPI, and implementing dynamic split-screen. All demos are written in GDScript; the dynamic split-screen demo also uses Godot shader language. The 3D scaling demo requires the Forward+ renderer; all others use Compatibility.

## GUI in 3D: forwarding input to a SubViewport

To make a 2D UI interactive when it lives on a 3D mesh, you need to convert 3D hit positions back into the SubViewport's 2D coordinate space and push the resulting event with `SubViewport.push_input()`. An `Area3D` node on the mesh detects whether the cursor is inside before forwarding events.

```gdscript
extends Node3D

var is_mouse_inside: bool = false
var last_event_pos2D := Vector2()
var last_event_time  := -1.0

@onready var node_viewport: SubViewport   = $SubViewport
@onready var node_quad:     MeshInstance3D = $Quad
@onready var node_area:     Area3D         = $Quad/Area3D

func _ready() -> void:
    node_area.mouse_entered.connect(_mouse_entered_area)
    node_area.mouse_exited.connect(_mouse_exited_area)
    node_area.input_event.connect(_mouse_input_event)

func _mouse_entered_area() -> void:
    is_mouse_inside = true
    node_viewport.notification(NOTIFICATION_VP_MOUSE_ENTER)

func _mouse_exited_area() -> void:
    node_viewport.notification(NOTIFICATION_VP_MOUSE_EXIT)
    is_mouse_inside = false

func _unhandled_input(input_event: InputEvent) -> void:
    # Pass non-mouse events (e.g. keyboard) directly to the viewport.
    for mouse_event in [InputEventMouseButton, InputEventMouseMotion,
                        InputEventScreenDrag, InputEventScreenTouch]:
        if is_instance_of(input_event, mouse_event):
            return  # Handled by physics picking instead.
    node_viewport.push_input(input_event)

func _mouse_input_event(_camera: Camera3D, input_event: InputEvent,
                        event_position: Vector3, _normal: Vector3,
                        _shape_idx: int) -> void:
    var quad_mesh_size: Vector2 = node_quad.mesh.size
    var now := Time.get_ticks_msec() / 1000.0

    # Convert world-space hit position to local quad space.
    var local_pos3D := node_quad.global_transform.affine_inverse() * event_position

    var event_pos2D := Vector2()
    if is_mouse_inside:
        # Map (-size/2 .. +size/2) → (0 .. viewport.size)
        event_pos2D.x = (local_pos3D.x / quad_mesh_size.x + 0.5) * node_viewport.size.x
        event_pos2D.y = (-local_pos3D.y / quad_mesh_size.y + 0.5) * node_viewport.size.y
    else:
        event_pos2D = last_event_pos2D

    input_event.position = event_pos2D
    if input_event is InputEventMouse:
        input_event.global_position = event_pos2D

    if input_event is InputEventMouseMotion or input_event is InputEventScreenDrag:
        input_event.relative = event_pos2D - last_event_pos2D
        input_event.velocity = input_event.relative / (now - last_event_time)

    last_event_pos2D = event_pos2D
    last_event_time  = now
    node_viewport.push_input(input_event)
```

## 2D in 3D: ViewportTexture on a mesh

Retrieve the texture from a SubViewport and assign it directly to a mesh's material albedo:

```gdscript
extends Node3D

func _ready() -> void:
    var viewport: SubViewport = $SubViewport
    # Render once into the viewport then stop clearing it.
    viewport.render_target_clear_mode = SubViewport.CLEAR_MODE_ONCE
    # Assign the live ViewportTexture as the quad's diffuse texture.
    $ViewportQuad.material_override.albedo_texture = viewport.get_texture()
```

The Pong game continues running in the SubViewport while the camera in the outer 3D scene orbits around the quad, creating the effect of a running game displayed on a TV screen.

## Dynamic split-screen

The dynamic split-screen demo uses two SubViewports—one per player camera—and composites them through a `ShaderMaterial` on a full-screen `TextureRect`. The camera controller updates shader parameters every frame to move the split line based on player positions.

```gdscript
extends Node3D

@export var max_separation            := 20.0
@export var split_line_thickness      := 3.0
@export var split_line_color          := Color.BLACK
@export var adaptive_split_line_thickness: bool = true

@onready var viewport1: SubViewport = $Viewport1
@onready var viewport2: SubViewport = $Viewport2
@onready var view:      TextureRect  = $View

func _ready() -> void:
    get_viewport().size_changed.connect(_on_size_changed)
    _on_size_changed()
    # Pass both viewport textures to the shader once.
    view.material.set_shader_parameter(&"viewport1", viewport1.get_texture())
    view.material.set_shader_parameter(&"viewport2", viewport2.get_texture())

func _process(_delta: float) -> void:
    _move_cameras()
    _update_splitscreen()

func _update_splitscreen() -> void:
    var screen_size  := get_viewport().get_visible_rect().size
    var p1_screen    := camera1.unproject_position(player1.position) / screen_size
    var p2_screen    := camera2.unproject_position(player2.position) / screen_size

    var thickness := _compute_thickness()
    view.material.set_shader_parameter(&"split_active",       _is_split_state())
    view.material.set_shader_parameter(&"player1_position",   p1_screen)
    view.material.set_shader_parameter(&"player2_position",   p2_screen)
    view.material.set_shader_parameter(&"split_line_thickness", thickness)
    view.material.set_shader_parameter(&"split_line_color",   split_line_color)

func _on_size_changed() -> void:
    var size := get_viewport().get_visible_rect().size
    viewport1.size = size
    viewport2.size = size
    view.material.set_shader_parameter(&"viewport_size", size)
```

The shader samples from `viewport1` or `viewport2` per pixel based on which side of the split line the pixel falls on. When both players are within `max_separation` units, `split_active` is `false` and the shader shows only `viewport1`'s full-screen output.

<Note>
Dynamic split screen (also called Voronoi split screen) was popularized by the Lego video games. The split line can take any angle—it is always perpendicular to the segment between the two players.
</Note>

## SubViewport size management

Always resize your SubViewports when the window size changes, otherwise the viewport texture becomes misaligned:

```gdscript
func _ready() -> void:
    get_viewport().size_changed.connect(_on_size_changed)

func _on_size_changed() -> void:
    var size := get_viewport().get_visible_rect().size
    $SubViewport.size = size
```

<Tip>
When you need a Viewport to render only once (for example, a pre-baked minimap or a static UI texture), set `render_target_update_mode` to `UPDATE_ONCE` after the first frame. This avoids re-rendering the texture every frame and can significantly reduce draw calls on mobile.
</Tip>
