Godot separates scene loading, data persistence, and background work into distinct APIs that compose cleanly. The seven demos in this group cover every pattern you are likely to need: global singletons via autoloads, non-blocking background loading with `ResourceLoader`, saving and restoring game state, switching scenes at runtime, serializing data to JSON and ConfigFile, and running work on background threads. All demos are written in GDScript; the threaded loading demo targets the Mobile renderer while the rest use Compatibility.

## Autoload: safe scene switching

The autoload demo registers `global.gd` as an autoload so any scene can call `Global.goto_scene()`. The key technique is deferring the actual scene swap to avoid freeing the current scene while code from that scene is still executing:

```gdscript
# global.gd  (registered as autoload named "Global")
extends Node

func goto_scene(path: String) -> void:
    # Defer the swap so the current scene can finish any ongoing callbacks.
    _deferred_goto_scene.call_deferred(path)

func _deferred_goto_scene(path: String) -> void:
    # Safe to free here—the deferred call guarantees no scene code is on the stack.
    get_tree().current_scene.free()

    var packed_scene: PackedScene = ResourceLoader.load(path)
    var instanced_scene := packed_scene.instantiate()

    get_tree().root.add_child(instanced_scene)
    # Must set current_scene AFTER adding to the tree.
    get_tree().current_scene = instanced_scene
```

## Threaded loading

`ResourceLoader.load_threaded_request()` starts a background load and returns immediately. You call `load_threaded_get()` when you actually need the resource—either blocking until complete or checking progress first with `load_threaded_get_status()`.

```gdscript
extends VBoxContainer

func _on_start_loading_pressed() -> void:
    # Issue all requests at once; they load in parallel on background threads.
    ResourceLoader.load_threaded_request("res://paintings/painting_babel.jpg")
    ResourceLoader.load_threaded_request("res://paintings/painting_las_meninas.png")
    ResourceLoader.load_threaded_request("res://paintings/painting_mona_lisa.jpg")
    ResourceLoader.load_threaded_request("res://paintings/painting_old_guitarist.jpg")
    ResourceLoader.load_threaded_request("res://paintings/painting_parasol.jpg")
    ResourceLoader.load_threaded_request("res://paintings/painting_the_swing.jpg")

    # Enable the "get loaded" buttons so the user can retrieve each painting.
    for btn: Button in $GetLoaded.get_children():
        btn.disabled = false

func _on_babel_pressed() -> void:
    # Blocks until the background load finishes (usually already done by now).
    $Paintings/Babel.texture = ResourceLoader.load_threaded_get(
        "res://paintings/painting_babel.jpg"
    )
    $GetLoaded/Babel.disabled = true
```

## JSON serialization

The serialization demo saves player position, health, rotation, and all active enemy positions to a JSON file. Because JSON does not natively support Godot types such as `Vector2`, the demo converts them with `var_to_str()` before writing and `str_to_var()` after reading.

```gdscript
extends Button

const SAVE_PATH = "user://save_json.json"

func save_game() -> void:
    var file := FileAccess.open(SAVE_PATH, FileAccess.WRITE)
    var player := get_node(player_node)

    var save_dict := {
        player = {
            # var_to_str converts Vector2/Vector3/etc. to a portable string.
            position = var_to_str(player.position),
            health   = var_to_str(player.health),
            rotation = var_to_str(player.sprite.rotation),
        },
        enemies = [],
    }

    for enemy in get_tree().get_nodes_in_group(&"enemy"):
        save_dict.enemies.push_back({ position = var_to_str(enemy.position) })

    file.store_line(JSON.stringify(save_dict))

func load_game() -> void:
    var file := FileAccess.open(SAVE_PATH, FileAccess.READ)
    var json := JSON.new()
    json.parse(file.get_line())
    var save_dict := json.get_data() as Dictionary

    var player := get_node(player_node) as Player
    player.position = str_to_var(save_dict.player.position)
    player.health   = str_to_var(save_dict.player.health)
    player.sprite.rotation = str_to_var(save_dict.player.rotation)

    # Clear existing enemies before spawning saved ones.
    get_tree().call_group(&"enemy", &"queue_free")

    var game := get_node(game_node)
    for enemy_config: Dictionary in save_dict.enemies:
        var enemy: Enemy = preload("res://enemy.tscn").instantiate()
        enemy.position = str_to_var(enemy_config.position)
        game.add_child(enemy)
```

## Runtime file loading

The `runtime_save_load` demo opens arbitrary files from disk at runtime—bypassing the Godot import pipeline entirely—and displays or re-exports them. It supports images (PNG, JPEG, WebP, SVG), audio (OGG, MP3, WAV), 3D scenes (GLTF, GLB, FBX), fonts (TTF, OTF, WOFF2), ZIP archives, and plain text.

