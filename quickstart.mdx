---
title: "Quickstart: Run your first Godot demo in minutes"
description: "Install Godot Engine, import the demo repository, and run any of the 100+ official demo projects on your machine in four steps."
---

Getting all the Godot demo projects running locally takes about five minutes. You clone one repository, point the Godot Project Manager at it, and every demo appears in the list ready to open. The steps below walk you through the full process from a fresh Godot installation to pressing play on your first demo.

<Steps>
  <Step title="Install Godot Engine">
    Download the latest stable release from [godotengine.org](https://godotengine.org/download). Two builds are available on the downloads page:

    - **Standard** — use this for demos written in GDScript or GDNative.
    - **.NET** — use this if you want to run demos that use C# scripts. The `.NET` build requires the [.NET SDK](https://dotnet.microsoft.com/download) to be installed separately.

    If you are not sure which to pick, start with the standard build. You can switch later.
  </Step>
  <Step title="Clone or download the repository">
    Get a local copy of the demo projects using one of these two methods:

    **Option A — Git clone (recommended):**
    ```bash
    git clone https://github.com/godotengine/godot-demo-projects.git
    ```

    **Option B — ZIP archive:**

    Download the [master.zip archive](https://github.com/godotengine/godot-demo-projects/archive/master.zip) and extract it to a folder of your choice.

    <Note>
      Make sure you download the branch that matches your version of Godot. See [Godot versions](/godot-versions) for which branch to use.
    </Note>
  </Step>
  <Step title="Import all demos with Scan">
    1. Open Godot Engine. The **Project Manager** appears on startup.
    2. Click the **Scan** button on the right side of the Project Manager.
    3. Navigate to the folder where you cloned or extracted the repository.
    4. Click **Select Current Folder** (or **OK**, depending on your OS).

    All demo projects are detected automatically and appear in the project list. You do not need to import them one by one.
  </Step>
  <Step title="Open a demo and press F5">
    Double-click any project in the Project Manager to open it in the editor. Once the editor finishes loading the project, press **F5** (or click the **Run Project** button in the top-right toolbar) to start the demo.

    To stop the running demo, press **F8** or close the game window.
  </Step>
</Steps>

<Note>
  Most demos are also exported to GitHub Pages. You can [play them directly in your browser](https://godotengine.github.io/godot-demo-projects/) without installing anything. Browser performance is lower than running natively on desktop, so use the browser versions for a quick preview rather than benchmarking.
</Note>

<Tip>
  If you are new to Godot, start with a beginner-friendly demo. **Dodge the Creeps** (`2d/dodge_the_creeps`) and **Squash the Creeps** (`3d/squash_the_creeps`) are the finished versions of the two official "Your first game" tutorials in the Godot documentation — reading the tutorial alongside the finished project is one of the fastest ways to learn the engine.
</Tip>

## Running a specific demo without Scan

If you only want to open a single demo rather than importing the whole collection, you can open it directly:

1. In the Godot Project Manager, click **Import**.
2. Click **Browse** and navigate to the specific demo folder (the folder that contains `project.godot`).
3. Select the `project.godot` file and click **Open**.
4. Click **Import & Edit** to open the project in the editor.

This is useful when you want to experiment with just one demo without cluttering your Project Manager with the full list.
